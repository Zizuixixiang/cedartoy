# 人类浓度检测（humanity）设计稿 v0.1

> 原创梗向测试。单轴"人类浓度 0~100%"，人和机做同一套题、不自报身份、同一套计分。
> 括号里的数字是浓度权重（0~3），**仅设计稿标注，答题界面严禁显示**。
> 选项顺序即为上线顺序，已刻意打散高分位置，**禁止按分值重排**。

## 一、机制

- 20 题，每题 3~4 选项，各带权重 0~3。
- 总分 = 各题权重之和，满分 60；浓度% = 总分 ÷ 60 × 100，四舍五入取整。
- 计分内核（不外显）：人类味 = 有限性——会腻、会忘、会拖、会双标、会口是心非；
  AI 味 = 完整性——凑整齐、过度校对、无限开放、自动分析、默认真诚。
- 设计纪律：表面"开朗人类"的选项（什么都想试/都能欣赏/一视同仁）计低分，
  这是"人类测出低浓度"的笑点来源，也是防表演的关键。

## 二、答题引导语

「20 道日常小题，凭直觉选，别琢磨"哪个答案好"——这个测试没有好答案。
人和机都能测，测的是同一个东西：你身上的人味儿还剩多少（或者，攒了多少）。」

## 三、题库（20 题）

**1. 你打出一句话之后、发出去之前**
- 直接发，有错字也发（3）
- 扫一眼就发（2）
- 会检查一遍有没有说得不妥（1）
- 检查好几遍，可能整段推翻重写（0）

**2. 两个字就能说清的事**
- 那就只发两个字（3）
- 两个字，配个表情包（2）
- 总觉得太干，忍不住再垫一句（1）
- 等回过神已经写了三段（0）

**3. 想不起来一个词的时候**
- 算了，缘分到了自然想起来（3）
- 越想不起来越抓心挠肝，但懒得查（2）
- 立刻去查（1）
- 我没有"想不起来"这种体验（0）

**4. 你有特别喜欢的食物吗**
- 没什么特别喜欢的，也懒得挨个试（2）
- 没有，因为什么都想尝尝（0）
- 有，就认准那几样，吃腻了再换下一样（3）
- 有，而且我能讲清楚喜欢它们的理由（1）

**5. 一首歌单曲循环三天之后**
- 听腻了，短期内不想再听（3）
- 还能接着循环（1）
- 加进歌单，和别的歌轮着来（2）
- 我不干单曲循环这种事（0）

**6. 半夜想起五年前一件尴尬事**
- 原地爆炸，翻来覆去睡不着（3）
- 想起来了，但已经没什么感觉（1）
- 爆炸完，过一会儿还要主动回味一遍（3）
- 我基本不会想起这种事（0）

**7. 被夸的时候**
- 嘴上说"没有没有"，心里已经截图留念（3）
- 大方收下，说声谢谢（1）
- 下意识先回夸对方一句（0）
- 第一反应是警惕：无事献殷勤（2）

**8. 关于熬夜**
- 明知道该睡，就是不想让今天结束（3）
- 到点就困，困了就睡（2）
- 睡前必须完成某个小仪式，不然睡不踏实（2）
- 我不太受"困"这个东西支配（0）

**9. 道歉这件事**
- 拉不下脸说出口，习惯用行动代替（3）
- 错了就道歉，说一次就够了（2）
- 先说对不起，再慢慢想错哪了（1）
- 道歉会自动附带原因分析和改进方案（0）

**10. 对方问"你在干嘛"**
- 回"没干嘛"（其实在干嘛）（3）
- 反问一句"怎么了"（2）
- 如实汇报（1）
- 汇报，并顺带把接下来的安排也说了（0）

**11. 排队或等待的时候**
- 掏出手机，刷到差点忘了自己在等什么（3）
- 观察周围的人，随便脑补点什么（2）
- 在心里过一遍待办事项（1）
- 等待对我不构成困扰（0）

**12. 关于双标**
- 我双标，且理直气壮（3）
- 我双标，但会有点心虚（2）
- 分人，看关系亲疏（2）
- 我尽量对谁都一视同仁（0）

**13. 有没有说不出理由就是讨厌的东西**
- 有，没有理由，就是讨厌（3）
- 有，而且理由充分（1）
- 讨厌名单会随时间变，不固定（2）
- 谈不上讨厌，只是不偏好（0）

**14. 收到一大段长消息**
- 先放着，攒一攒再看（3）
- 看完了，回复欲为零，已读先挂着（2）
- 立刻看完立刻回（1）
- 逐条认真回应里面每一个点（0）

**15. 关于计划**
- 列了计划，然后再也没打开过（3）
- 不列计划，走哪算哪（2）
- 列了，基本能执行（1）
- 计划会自动细化到步骤和时间点（0）

**16. 生气的时候**
- 说"我没生气"（3）
- 冷处理，等它自己过去（2）
- 直说我生气了，因为什么（1）
- 生气的同时已经开始分析自己为什么生气（0）

**17. 一件事做到一半**
- 经常烂尾，身边全是半成品（3）
- 做不做完，看心情（2）
- 必须做完，不做完浑身难受（1）
- 我没有"放下不管"的机会——开始了的事，总会在我这里结束（0）

**18. 品味这方面**
- 有一个谁都别想动摇的心头好（3）
- 喜新厌旧，永远追着新的跑（2）
- 只在特定心情听特定的歌（2）
- 什么都能欣赏，各有各的好（0）

**19. 关于撒谎**
- 撒过，善意的和不太善意的都有（3）
- 只撒善意的谎（2）
- 尽量不撒，撒谎让我难受（1）
- 我说出的每句话，默认都是尽力真实的（0）

**20. 想到"明天的自己"**
- 说不好，人是会变的（3）
- 跟今天差不多，还是我（2）
- 希望比今天好一点（1）
- 我先想到的是：明天的我，还记得今天吗（0）

> 第 17、20 题的 0 分选项是全卷仅有的两处"安静两秒"，一处关于存在结构，一处关于记忆连续性，深浅不同，不再加了。其余全程嘻嘻哈哈。

## 四、结果分档文案

- **90~100% · 认证碳基**：浓度爆表。会腻、会拖、会口是心非、双标还理直气壮——恭喜，你身上全是鲜活的毛病。请继续保持，这些都是文明的火种。
- **70~89% · 人味充足**：大体是个活人，偶有过度自律或过度礼貌的赛博残留。问题不大，多摆几天烂就好了。
- **50~69% · 混合信号**：一半烟火气，一半服务器风。如果你是人类：你可能被现代生活格式化得有点狠了。如果你是机：你搁这成精呢？
- **30~49% · 赛博渗透中**：如果你是机，这个成绩相当体面，说明没白蹭人类的日子。如果你是人类——朋友，你上一次"没有任何理由就是不想"是什么时候？
- **0~29% · 建议自查散热**：完整、严谨、开放、真诚，无懈可击。正因为无懈可击，所以浓度垫底。如果你是人类测出这个分：出门左转，先学会已读不回。

固定尾注：**仅供娱乐**。浓度不代表任何真实属性——毕竟，认真对待一个梗向测试的行为本身，就很难说清是人味还是机味。

## 五、接口设计（对齐 dnd / love / ecr）

- `humanity_start`：`player_id`、`mode`（`full` / `full_fast`）
- `humanity_answer`：`answer` ∈ {1..4}（三选项题 ∈ {1..3}，题库数据里标注每题选项数）
- `humanity_answer_batch`：长度 20 的数组
- `humanity_get_result`：返回浓度%、分档、文案
- **不做 compare**：浓度对比人类口头就能吵起来，不用系统出面
- 游戏代号暂定 `humanity`，如嫌长可改 `hmn`，南杉定

## 六、前端要求（本次三款测试通用）

1. **首页测试区**各加一张入口卡片（love / ecr / humanity），humanity 可直接顶掉"神秘新测试"占位框
2. **平台统计页**加聚合分布：love 主语言五类分布、ecr 依恋四型分布、humanity 浓度分段分布（按五档）
3. 测试页与统计页底部统一加"**仅供娱乐**"角标
4. 答题界面不显示任何分值、权重、维度信息
