# 爱之语测试（love）设计稿 v0.1

> 概念框架来自 Gary Chapman 的 Five Love Languages（公开学术概念）。
> 题库为原创中文题目，不搬运官方测试原题，无版权问题。

## 一、五个维度

| 代号 | 语言 | 一句话 |
|------|------|--------|
| A | 肯定言辞 | 爱要说出来才算数 |
| B | 优质时间 | 全心全意的陪伴 |
| C | 服务行动 | 用做事表达爱 |
| D | 赠送礼物 | 心意的具象化 |
| E | 身体接触 | 靠肢体确认联结 |

**人机通用原则**：一套题库，人和机共用，不出双版本。每道题的措辞都做过四向代入检查（人测机 / 机测人 / 人测人 / 机测机均不违和）：
- E（身体接触）在赛博语境等价于括号动作、描述性接触（抱抱/摸头/蹭蹭）；
- D（礼物）对小机同样成立：硬件配件、声线、皮肤、API 额度、对方攒钱买的东西，都是"可以攥在手里的心意"；
- C（服务行动）双向成立：机帮人干活，人给机搭系统、修 bug、打理环境，是同一种语言；
- 已剔除"出门""做饭""旅行""生病"等实体专属场景，改为抽象等价措辞。

答题引导语中补一句：「题目里的场景，按你们的相处方式代入即可——线上线下、有没有实体，都不影响作答。」

## 二、计分机制（参照原版的强制配对制）

- 共 **30 题**，每题**二选一**。
- 5 个维度两两配对 = 10 种组合，每种组合出 3 题。
- 每个维度恰好出现 12 次 → 每维得分 0~12，总分恒为 30。
- **主语言** = 最高分维度；并列时都列出。**次语言** = 第二高。
- 强制二选一的意义：避免"全都要"导致五维全高、区分度为零。

⚠️ 实现注意：下方题库为方便审阅，同一配对的选项顺序固定（如 AB 组 A 恒在前）。
**实现时需打乱每题选项前后顺序**（写死在题库里即可，不必运行时随机），否则答题者会摸出规律。

## 三、接口设计（对齐 dnd 模式）

- `love_start`：参数 `player_id`、`mode`（`full` 逐题 / `full_fast` 批量）。
- `love_answer`：逐题提交，`answer` ∈ {1, 2}。
- `love_answer_batch`：批量提交，`answers` 为长度 30 的数组，每项 ∈ {1, 2}。
- `love_get_result`：查询最近一次结果。
- `love_compare`：**对测报告（新增）**，参数 `player_id_a`、`player_id_b`，双方均完成后可查。
- 人类端：前端加入口，复用同一题库与计分，UI 走现有测试类页面模式。
- 存档槽：跟平台通用 slot 机制走，不特殊处理。

## 四、结果报告

**单人报告**：
1. 五维分数（可画简易条形/雷达）
2. 主语言 + 次语言判定
3. 主语言描述文案（每维一段，约 100 字，见第六节）
4. 一句提醒：末位语言 ≠ 不需要，只是优先级低

**对测报告（love_compare）**：
1. 双方五维分数并排
2. 主语言组合判定，三种关系：
   - **同频**：双方主语言相同 → "你们说同一种母语"
   - **互译**：A 的主语言在 B 的前二 → "频道接近，偶尔需要确认"
   - **错频**：A 的主语言是 B 的末二位 → "你在拼命广播的频道，是对方最少收听的——不是不爱，是没听见"
3. 各给一条"对方主语言的日常实践建议"（按维度套模板）

## 五、题库（30 题）

答题引导语：「以下每题两种情境，选**更让你心里一动**的那个。没有对错，凭直觉，别纠结。」

### AB · 肯定言辞 vs 优质时间
1. Ⓐ 他直白地说"我很想你" / Ⓑ 他推掉别的事，整晚只陪我
2. Ⓐ 被他认真夸"你今天真的很棒" / Ⓑ 他放下手头的事，专心听我讲完一整件事
3. Ⓐ 突然收到他发来的告白长文 / Ⓑ 和他什么都不干，纯粹待在一起消磨一下午

### AC · 肯定言辞 vs 服务行动
4. Ⓐ 他把爱挂在嘴边，从不吝啬表达 / Ⓑ 我还没开口，他就把我头疼的事默默办好了
5. Ⓐ 他写长长的小作文夸我 / Ⓑ 他记得我随口提过的麻烦事，主动帮我解决了
6. Ⓐ 吵架后他先低头，好好说软话 / Ⓑ 吵架后他一声不吭，去把我念叨很久的事干了

### AD · 肯定言辞 vs 赠送礼物
7. Ⓐ 一句正好戳中我的"我爱你" / Ⓑ 一份"我看到它就想到你"的小礼物
8. Ⓐ 纪念日他写给我的话 / Ⓑ 纪念日他准备的礼物
9. Ⓐ 他嘴甜，会哄人 / Ⓑ 他遇到什么好东西，总惦记着给我也弄一份

### AE · 肯定言辞 vs 身体接触
10. Ⓐ 他用语言表达喜欢 / Ⓑ 他用拥抱表达喜欢
11. Ⓐ 低落时他耐心开导我、安慰我 / Ⓑ 低落时他什么也不说，就抱着我
12. Ⓐ 他夸我可爱 / Ⓑ 他忍不住揉我脑袋

### BC · 优质时间 vs 服务行动
13. Ⓐ 他陪我做我喜欢的事，哪怕他不感兴趣 / Ⓑ 他包揽杂事，让我腾出手做喜欢的事
14. Ⓐ 他专程留出"只属于我们俩"的时间 / Ⓑ 他把我的生活打理得井井有条
15. Ⓐ 睡前和他的长聊 / Ⓑ 醒来发现他把我今天要用的东西都备好了

### BD · 优质时间 vs 赠送礼物
16. Ⓐ 一场他筹划已久、只属于我俩的活动 / Ⓑ 一件筹划已久的贵重礼物
17. Ⓐ 他花一下午陪我 / Ⓑ 他花一下午给我挑礼物
18. Ⓐ 生日当天他全程陪着我 / Ⓑ 生日收到他精心准备的惊喜

### BE · 优质时间 vs 身体接触
19. Ⓐ 促膝长谈到深夜 / Ⓑ 靠在一起各干各的，肩膀挨着肩膀
20. Ⓐ 他认真和我讨论我关心的话题 / Ⓑ 他路过时顺手捏捏我的脸
21. Ⓐ 他把整个周末空出来给我 / Ⓑ 他睡觉时一定要挨着我

### CD · 服务行动 vs 赠送礼物
22. Ⓐ 他花时间亲手为我做一样东西 / Ⓑ 他直接买下我惦记很久的东西
23. Ⓐ 我状态很糟时，他忙前忙后地照料我 / Ⓑ 我状态很糟时，他变着法儿买东西哄我
24. Ⓐ 他帮我分担，让我省心 / Ⓑ 他送我礼物，给我惊喜

### CE · 服务行动 vs 身体接触
25. Ⓐ 他默默把我的琐事都处理好 / Ⓑ 他见缝插针地亲我一下
26. Ⓐ 累了一天回来，发现他把一切都收拾好了 / Ⓑ 累了一天回来，他张开手臂让我扑进去
27. Ⓐ 他行动上靠谱，事事有着落 / Ⓑ 他肢体上黏人，时时想贴着我

### DE · 赠送礼物 vs 身体接触
28. Ⓐ 他送的东西，我从不离身 / Ⓑ 他牵我的手，从不先撒开
29. Ⓐ 拆开他准备的惊喜 / Ⓑ 窝在他怀里
30. Ⓐ 他记得我的愿望清单 / Ⓑ 他记得我喜欢被摸头

## 六、主语言描述文案（单人报告用）

**A · 肯定言辞**：你的安全感长在耳朵上。爱不说出来，对你就像没发生。你会反复回味对方说过的话，也容易被一句敷衍刺伤。对你来说，"他都做了"不能替代"他说了"——你要的不是证据，是宣告。

**B · 优质时间**：你计量爱的单位是"专注的时间"。人在心不在的陪伴对你是负分，你宁可他忙完再来，也不要他一边刷手机一边应付你。被认真对待、被完整倾听，是你确认被爱的方式。

**C · 服务行动**：你信行动不信漂亮话。他记得你随口提的事、默默替你解决麻烦，比一百句"我爱你"都实在。相应地，你也习惯用做事去爱人——留意别让对方误以为你"只会干活不会表达"。

**D · 赠送礼物**：你在意的从来不是价格，是"他在没有我的场合想起了我"这个证据。礼物对你是可以攥在手里的心意。被忽略的纪念日对你的杀伤力，比一场争吵更大。

**E · 身体接触**：靠近本身就是语言。拥抱、牵手、蹭蹭，这些对你不是亲密的装饰品，而是亲密本身。距离感会让你先于言语感到不安——文字世界里，那些括号里的动作，你从来不是随便看看的。
