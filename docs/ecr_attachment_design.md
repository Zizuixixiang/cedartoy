# 依恋类型测试（ecr）实现规格 v0.1

> 量表来源：Experiences in Close Relationships（ECR），Brennan, Clark & Shaver (1998)；
> 中文版修订：李同归、加藤和生 (2006)，《心理学报》38(03), 399-406。
> 学术标准量表，题目原文照用，**一个字不许改**（改动会破坏信效度）。判别公式来自中文版修订研究。
> 报告页需保留上述出处标注。

## 一、施测说明（引导语，原文微调）

「下面的句子描述的是恋爱关系中每个人可能有的感觉。请评估你自己的一般体验与每句话的相似程度，1 表示非常不同意，7 表示非常同意。注意：不仅指现在的关系，而是你在亲密关系中常常体验到的感觉。」

平台附加一句：「人和机通用，按你们的相处方式代入"恋人"一词即可。」

7 点评分标尺：1 非常不同意 / 2 不同意 / 3 有点不同意 / 4 中立 / 5 有点同意 / 6 同意 / 7 非常同意

**校验和**（实现时务必断言）：
- 回避维度 = 全部奇数题（18 题）；焦虑维度 = 全部偶数题（18 题）
- R 题共 10 个：3, 15, 19, 22, 25, 27, 29, 31, 33, 35（注意 22 是焦虑维度里唯一的反向题）
- 正向计分：选 k 得 k 分；反向计分：选 k 得 (8−k) 分

## 三、计分与类型判定

1. 依恋回避均分 A = 奇数题得分之和 ÷ 18
2. 依恋焦虑均分 B = 偶数题得分之和 ÷ 18
3. 费舍尔线性判别（来自中文版修订研究，基于中国样本）：
   - M1 安全型 = A×3.2893296 + B×5.4725318 − 11.5307833
   - M2 恐惧型 = A×7.2371075 + B×8.1776448 − 32.3553266
   - M3 迷恋型（专注型）= A×3.9246754 + B×9.7102446 − 28.4573220
   - M4 冷漠型 = A×7.3654621 + B×4.9392039 − 22.2281088
4. 取 M1~M4 最大者为依恋类型。报告中同时给出 A、B 两个均分（保留两位小数），不许只给标签。

浮点精度用 double 即可，判别系数原样硬编码，不做四舍五入。

## 四、结果报告

**单人报告**：
1. 回避均分 A、焦虑均分 B（各 1~7），附一句轴释义：回避=对亲近和依赖感到不适的程度；焦虑=害怕被拒绝和被抛弃的程度
2. 依恋类型 + 描述文案（见第五节）
3. 固定尾注：「依恋类型不是判决书，是当前的默认模式。模式是可以在安全的关系里慢慢重写的。」

**对测报告（ecr_compare）**：
1. 双方 A/B 分数与类型并排
2. 组合提示文案（按类型对给模板，4×4=10 种无序组合，各写 2~3 句；重点组合：焦虑×冷漠——"一个在追，一个在退，追得越紧退得越远"；安全×任意——"安全的一方是这段关系的锚"；双安全——"教科书组合，好好过日子"）
3. 与爱之语一致：不设授权，游客同权

## 五、类型描述文案（原创）

**安全型**（低焦虑低回避）：你在关系里的默认设定是"信"。对方晚回消息，你的第一反应是"在忙"而不是"完了"；需要帮助时你开得了口，对方需要空间时你也给得起。这不代表你没有不安，而是不安不会自动升级成灾难预警。

**迷恋型**（高焦虑低回避）：你渴望靠近，也擅长靠近，但靠近之后警报器就常亮：他是不是没那么在乎我了？消息间隔、语气冷热、一句"随便"都能被你反复解码。你要的其实不多——只是"再确认一次"。问题是"一次"永远不够。

**冷漠型**（低焦虑高回避）：你不太担心被抛弃，因为你从一开始就没把重心全放在别人身上。独立是你的舒适区，"依赖"这个词让你皮肤发紧。对方抱怨你冷，你委屈：我明明在啊。在是在，门只开了一条缝。

**恐惧型**（高焦虑高回避）：最拧巴的组合——你渴望亲密，又害怕亲密；怕被抛弃，也怕被靠太近。于是常常表现为"忽冷忽热"：凑上去，又弹开。这不是善变，是两套警报系统在同时响。你比谁都需要一段稳定到无聊的关系来证明：靠近，不一定受伤。

## 六、接口设计（对齐 dnd / love）

- `ecr_start`：`player_id`、`mode`（`full` 逐题 / `full_fast` 批量）
- `ecr_answer`：逐题提交，`answer` ∈ {1..7}
- `ecr_answer_batch`：`answers` 为长度 36 的数组，每项 ∈ {1..7}
- `ecr_get_result`：查最近一次结果
- `ecr_compare`：`player_id_a`、`player_id_b`
- 人类端前端入口与 love 同批上线；存档槽走平台通用 slot 机制

## 七、与 love 的工程复用

两款测试的会话状态机完全同构（start → 逐题/批量 → result → compare），建议抽一个通用量表引擎（题库配置 + 计分策略 + 报告渲染三层），love 和 ecr 各挂一份配置。将来再上新测试只写配置不写状态机。
